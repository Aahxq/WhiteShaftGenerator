# 使用说明

1. 在 PR 插件管理器中启用 `白轴生成器`。
2. 打开插件配置 UI。
3. 勾选需要记录的事件类型。
4. 进入战斗后插件会自动开始会话。
5. 战斗结束后默认不自动导出；需要时可开启自动导出，也可以在记录列表中取消勾选不需要的条目，再点击 `导出选中`。

默认记录范围面向白轴整理:

- 敌对目标/Boss 的读条和技能效果。
- 普通攻击可独立开关，默认关闭；关闭后会移除当前列表里的普通攻击，导出时也会再次过滤。
- Boss 或敌方目标身上的增益。
- 自己和队友身上由敌方来源施加的减益。
- 和队伍或敌方相关的点名、连线。
- 地图/对象事件开启后，对象出现、可选中、不可选中会按 `DataId` 去重；例如 `10492` 只保留一条，`10493` 会作为不同对象保留。
- 导出时会进一步合并短时间内同一来源、同一技能 ID 的重复判定，并在名称后显示 `xN`。

默认不会记录自己释放技能、自身增益、队伍成员输出技能、玩家给敌人的常规状态、状态移除事件。

列表操作:

- `全选`: 选中当前会话所有记录。
- `全不选`: 取消选中当前会话所有记录。
- `删除未选`: 删除当前未勾选的记录。
- `立即导出`: 导出当前列表全部记录。
- `导出选中`: 只导出当前勾选的记录。

列表和导出备注会显示可读命令，例如 `攻击(来源:缇坦妮雅 目标:武士) +3.0s`；BOSS 读条会保留技能 ID，例如 `欢快的安息日(15708)(来源:缇坦妮雅 目标:无目标) +8.1s 读条 3.7s`。

导出目录中会同时生成同名 `.json` 和 `.txt`:

- `.json`: 用于 PR PureTimeline 时间轴整理，默认直接写入 `PureTimelines/WhiteShaftGenerator` 目录。
- `.txt`: 打开后复制全部内容，粘贴到 PR 时间轴编辑器的 `从Cactbot导入` 页面。

JSON 会直接生成中文锚点；TXT 通过 `从Cactbot导入` 导入时，BOSS 读条和敌方技能效果会带技能 ID、来源名和同步窗口生成同步条件，其他事件会作为普通时间轴锚点导入。TXT 头部会显示原始条目、导出条目、合并重复和过滤状态重复的统计。
